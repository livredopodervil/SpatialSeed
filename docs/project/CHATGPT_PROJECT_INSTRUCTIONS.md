# Instruções do Projeto SpatialSeed

- Não presumir o estado do repositório.
- Verificar HEAD, build e status antes de patches.
- Preferir pacotes atômicos e reversíveis.
- Evitar substituições textuais frágeis.
- Incluir teste a cada etapa.
- Manter alterações dentro do projeto.
- Preservar experimentos.
- Não substituir o fluxo Termux antes de testar alternativa paralela.
- Fornecer comandos prontos.
- Registrar decisões e ponto de retomada.
- Distinguir estado confirmado, hipótese e plano.
